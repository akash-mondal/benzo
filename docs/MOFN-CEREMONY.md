# Sound M-of-N dual-control — status + the production ceremony

## What is DONE and verified (sound, on-chain)

An **org note** is a shielded note owned by an M-of-N member set, not a single key.
Its `recipient_pk` is preimage-bound:

```
recipient_pk = Poseidon2(memberRoot, threshold, akGroupPub ; ORG_NOTE_DOMAIN=0x09)
```

Because that's a hash preimage no single key can satisfy, an org note can **only** be
spent through `pool.transfer_org`, whose proof (`joinsplit_org`, VK id `JSPLITORG`)
enforces **≥ threshold distinct member EdDSA signatures in-circuit**, with an
unlinkable org nullifier (`Poseidon2(Poseidon2(akGroup,blinding;0x07), leafIndex; 0x02)`).

**SDK primitives** (`@benzo/core/notes.ts`): `orgRecipientPk`, `orgNullifier`,
`akGroupPub`. **Witness builder + on-chain proof** exercised by
`tests/e2e/joinsplit-org-onchain.mjs`, verified live on testnet:

- a 2-of-3 org-note transfer → `verify_proof(JSPLITORG) => true`
- a tampered public input → `false` (fail-closed)
- a **sub-threshold (1-of-2) spend cannot even produce a proof** — the circuit's
  threshold constraint fails at witness generation (`OrgInputAuth line 128`)

⇒ Org funds are unspendable on a single key — enforced **in-circuit and on-chain**, not by a server.
The `pool.transfer_org` settle path is covered by the contract unit test
`contracts/pool/src/test.rs::transfer_org_settles_under_jsplitorg_vk_and_is_vk_gated`
**and by a full real-USDC settle on live testnet** (`tests/e2e/joinsplit-org-settle-onchain.mjs`):
shield 0.2 USDC into an `orgRecipientPk` note → `pool.transfer_org` under a 2-of-3 quorum
(org nullifier recorded, `next_index` +2) → withdraw one output to a public account (real USDC
exits, 16.1 → 16.2). Tx: shield `a0143f40…`, transfer_org `850fa1cc…`, withdraw `27df789e…`
(against the TEE-rotated JSPLITORG VK). So real money makes a full round-trip *through* M-of-N
dual control, not just a `verify_proof` of life.

## The trusted setup — DONE (TEE-attested phase-2, cut over on-chain)

The `JSPLITORG` proving key is **no longer a single-author dev key**. It was
re-contributed in a **phase-2 ceremony run inside a Phala dstack / Intel TDX enclave**,
where the contribution entropy (the "toxic waste") was generated by the in-enclave
CSPRNG and destroyed inside the sealed enclave — never written out or returned. The
enclave returns a **TDX attestation quote** whose `report_data` binds
`sha256(inputZkeyHash) ‖ sha256(outputZkeyHash)`, so anyone can verify the new key was
produced inside *that attested enclave* for *that exact zkey transition*. A solo
operator therefore gets a credibly **1-of-N-honest** setup: the attested enclave is the
independent honest party, and no human ever held the toxic waste.

> **Why this was the only "multi-party" concern.** The M-of-N *signers* are not a
> multi-party problem — "M of N" = M distinct member keys signing the same spend, all
> holdable by one operator (the circuit only checks M distinct members of the on-chain
> `member_root` signed). Only the phase-2 *ceremony* wanted independence, and the TEE
> attestation supplies it.

### How it was produced + cut over (reproducible)

```bash
# 1) deploy an ephemeral TEE ceremony CVM (v4 enclave image: /contribute + /artifact + baked zkey)
phala deploy --name benzo-ceremony --compose deploy/phala/docker-compose.ceremony.yml \
  --instance-type tdx.large --wait --json

# 2) run the in-enclave contribution → fetch the new zkey → export VK → write transcript+quote
BENZO_ENCLAVE_URL=https://<app-id>-8080.<node>.phala.network \
  node scripts/tee-ceremony.mjs joinsplit_org

# 3) verify the TDX quote against live Intel collateral + the report_data binding + the on-disk zkey
node scripts/verify-ceremony-quote.mjs joinsplit_org
#   => status UpToDate; report_data binds sha256(inHash)||sha256(outHash); .tee.zkey intact

# 4) promote the TEE pair to canonical (with .dev.zkey.bak backup) + provenance copy in ceremony/
#    then rotate the on-chain VK (set_vk is one-time; rotate_vk overwrites, admin-gated):
stellar contract invoke --id <verifier> --source benzo-deployer \
  --rpc-url "$SOROBAN_RPC_URL" --network-passphrase "$NETWORK_PASSPHRASE" -- \
  rotate_vk --vk_id JSPLITORG --vk "$(node scripts/groth16-to-soroban.mjs vk circuits/build/joinsplit_org/joinsplit_org_vk.json)"

# 5) re-prove against the production key + confirm it still verifies on-chain
set -a; . ./.env; set +a
node tests/e2e/joinsplit-org-onchain.mjs   # JSPLITORG => true; tampered => false; sub-threshold unprovable
```

Nothing else changed — `pool.transfer_org`, the circuit, and the SDK are unchanged;
only the zkey/VK behind `JSPLITORG` was swapped. **Mainnet** is then an env swap
(Protocol 26, `deployments/mainnet.json`) plus this same TEE-attested run (or a
multi-party run with real external contributors) against the mainnet verifier.
